## How to Use This File

Each entity has a **status tier** and a list of **source files**. Load source files when the entity is directly relevant to the current session. For Background and Archived entries, the summary here is usually sufficient — pull the sources only if something has specifically resurface them.

**Status tiers:**
- 🔴 **Active** — in play, unresolved threads, load sources on demand
- 🟡 **Background** — established, stable, not currently driving story
- ⚪ **Archived** — resolved or dormant; one-line reminder only

**Cross-references** are only included when the connection is non-obvious and both directions benefit from knowing it. They explain the link; they don't just point.

**Maintenance**: After each session, add new entities, update statuses, and flag candidates for promotion or demotion in the `REVIEW QUEUE` section at the bottom. Status changes require player confirmation — the DM flags, the player decides.
